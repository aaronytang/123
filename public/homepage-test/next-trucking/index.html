Influence

One design language for every Next product — an initiative I started

No one asked for a design system; shipping without one was just slower every sprint. I partnered with engineering to build a unified system now adopted across all Next web and mobile products — every team ships faster from the same parts.

Strategy

Design foundation for a system that mostly runs itself

In an automated dispatch system, design doesn't serve human tasks — it serves exception management, the moments automation hands back to a person. The architecture below is built around that inversion.

The best interface for an automated process is almost none.

Approach

Better experience through less UI

Each removed input is an error that can't happen and a minute a dispatcher gets back. The work was deleting steps from the process itself, not polishing the steps that shouldn't exist.

Outcome

Exceptions surfaced with the data to act — $5 saved per container, 25% more throughput

When automation needs a human, the system presents the exception with the context and the action — not a dashboard to dig through. That single design decision carries the business numbers on this page.

Strategic trade-offs

What we chose — and what we deliberately didn't build

  • 01
    Stopped optimizing human task speed. Polishing workflows that automation would absorb within quarters was sunk cost. Investment went to the exception layer instead — the part that gets more valuable as automation grows.
  • 02
    No per-role dashboards. Dashboards invite monitoring; exceptions demand action. We built actionable queues over passive overviews, even though dashboards were the requested feature.
  • 03
    Design system before new features. A quarter of slower feature delivery bought permanently faster delivery for every team after. I made that case to engineering leadership and it held.

Post-launch

The numbers that mattered to the business

25% throughput lift and $5 loss reduction per container at freight volume. The design system continues to compound — every new Next product starts from shared, proven parts instead of a blank file.
